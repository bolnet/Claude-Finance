Create `.claude/commands/demo.md` with the following structure:

**Frontmatter:**
```yaml
---
description: Guided walkthrough of all Finance AI Skill capabilities — 11 tools, 2 personas
allowed-tools: Bash(python3:*), Bash(pip:*), Write, Read, mcp__finance__ping, mcp__finance__validate_environment, mcp__finance__analyze_stock, mcp__finance__get_returns, mcp__finance__get_volatility, mcp__finance__get_risk_metrics, mcp__finance__compare_tickers, mcp__finance__correlation_map, mcp__finance__ingest_csv, mcp__finance__liquidity_predictor, mcp__finance__predict_liquidity, mcp__finance__investor_classifier, mcp__finance__classify_investor
model: sonnet
---
```

No `argument-hint` — the demo takes no arguments.

**Body structure (this IS the prompt Claude receives):**

1. **Welcome Section** — Introduce the Finance AI Skill. Explain it has 11 tools across 3 categories (market analysis, ML workflows, environment). State that the walkthrough will demonstrate each tool with real data and plain-English explanations. Mention approximate duration (~5-10 minutes).

2. **Environment Context** — Same !-injection block as finance.md:
   - Python version, finance packages check, working directory, output directory status

3. **Walkthrough Instructions** — Critical behavioral instructions for Claude:
   - "You are running a guided demo. Execute each step below IN ORDER."
   - "After each step: (a) run the tool/code, (b) show the result, (c) provide a PLAIN-ENGLISH EXPLANATION of what just happened in 2-3 sentences, (d) print a separator line, (e) then proceed to the next step."
   - "Do NOT ask the user for input between steps — this is a self-running walkthrough."
   - "If any step fails (network error, missing package), explain the error in plain English and continue to the next step."

4. **Step Definitions** — Each step as a numbered section:

   **Step 1: Environment Check**
   - Call `validate_environment` MCP tool
   - Explain: confirms all required packages are installed

   **Step 2: Health Check**
   - Call `ping` MCP tool
   - Explain: verifies the MCP server is running and responsive

   **Step 3: Price Chart (analyze_stock)**
   - Analyze AAPL for the last 90 days
   - Explain: fetches real-time Yahoo Finance data, produces a price chart

   **Step 4: Returns Analysis (get_returns)**
   - Get returns for AAPL, last 90 days
   - Explain: calculates daily and cumulative returns, shows how $1 invested grew

   **Step 5: Volatility (get_volatility)**
   - Get volatility for AAPL, last 90 days
   - Explain: measures price variability, 21-day rolling window shows stability trends

   **Step 6: Risk Metrics (get_risk_metrics)**
   - Get risk metrics for AAPL, last 1 year
   - Explain: Sharpe ratio (risk-adjusted return quality), max drawdown (worst peak-to-trough), beta (market sensitivity)

   **Step 7: Ticker Comparison (compare_tickers)**
   - Compare AAPL vs MSFT, last 90 days
   - Explain: normalizes both to $100 start for fair visual comparison

   **Step 8: Correlation Heatmap (correlation_map)**
   - Correlation map for AAPL, MSFT, GOOGL, last 90 days
   - Explain: shows how closely stock returns move together; implications for diversification

   **Step 9: CSV Ingestion (ingest_csv)**
   - Use the bundled demo CSV (this will be created in Phase 7; for now, reference `demo/sample_portfolio.csv`)
   - Explain: automatic column detection, summary statistics, data cleaning
   - NOTE in the command: "If demo/sample_portfolio.csv does not exist yet, skip this step with a note that the sample CSV will be added in a future update."

   **Step 10: ML Regression (liquidity_predictor + predict_liquidity)**
   - Same CSV dependency note as Step 9
   - Train the liquidity model on demo CSV, then predict for a sample client
   - Explain: trains a regression model, evaluates with R-squared and RMSE, then scores a new observation

   **Step 11: ML Classification (investor_classifier + classify_investor)**
   - Same CSV dependency note as Steps 9-10
   - Train investor classifier on demo CSV, then classify a sample investor
   - Explain: trains a classification model with cross-validation, then assigns a segment label to a new investor

5. **Persona Showcase** — After the 11 tools:
   - Brief note: "The skill also includes two persona modes that frame analysis differently"
   - Describe /finance-analyst (equity analyst lens) and /finance-pm (portfolio manager lens)
   - Note: "Run /finance-analyst or /finance-pm to try them"

6. **Completion Summary** — Final section:
   - Print a formatted summary listing all 11 tools demonstrated with one-line descriptions
   - Group by category: Environment (2), Market Analysis (6), ML Workflows (3)
   - State: "Demo complete. You can now use /finance for ad-hoc analysis, or /finance-analyst and /finance-pm for role-specific framing."
   - End with the standard disclaimer

The tone should be instructional but not verbose. Each step explanation should be 2-3 sentences maximum.
